# スプレッドシート連携の設定手順

このアプリは、Googleスプレッドシートを公開URLとして読み込み、記載されているお店情報をアプリに反映できます。
設定が完了すると、スプレッドシートを編集してアプリを再読み込みするだけで内容が反映されます(コード操作は不要)。

## 1. スプレッドシートを作る

1. [docs/busan-sheet-template.csv](busan-sheet-template.csv) を開き、内容をダウンロード(またはコピー)する
2. Google スプレッドシートを新規作成し、「ファイル」→「インポート」→上記CSVをアップロードして読み込む
   (既存のシートに読み込む場合は「現在のシートを置換」を選択)
3. 1行目はヘッダー行(`visited,region1,line,...`)なので、そのまま残しておく

## 2. 「ウェブに公開」する

1. スプレッドシートのメニューから「ファイル」→「共有」→「ウェブに公開」
2. 公開する範囲で、対象のシート(タブ)を選択
3. 形式を「カンマ区切りの値(.csv)」に変更
4. 「公開」を押す
5. 表示されたURL(`https://docs.google.com/spreadsheets/d/e/.../pub?output=csv` のような形式)をコピーする

このURLを私(Claude)に伝えてください。`js/app.js` 内の `SHEET_CSV_URL` にこの値を設定すれば連携が有効になります。

## 3. 列の意味と編集できる項目

| 列名 | 内容 | 編集 |
|---|---|---|
| visited | 訪問済みなら `TRUE` | ○ |
| region1 | 地域1(釜山固定) | - |
| line | 路線(例: 1号線) | ○ |
| area_ja | エリア・駅名(日本語) | ○ |
| area_ko | エリア・駅名(韓国語) | ○ |
| name_ko | 店名(韓国語)※これが一致キーになります | ○(既存店の名前は変更しないでください) |
| naver_url | Naver地図リンク | ○ |
| google_url | Google地図リンク(空欄でも自動生成されます) | ○ |
| address_road / address_lot | 住所 | ○ |
| menu | 代表メニュー | ○ |
| source | 出典・番組名 | ○ |
| notes | 特記事項 | ○ |

**店名(name_ko)は、既存のお店データと突き合わせる際の「キー」として使っています。既存店の店名は変更しないでください**(変更すると別の新しい店として扱われます)。

## 4. できること・できないこと

- ✅ **既存店の情報編集**: メニュー・住所・URL・特記・訪問済みなど、即座にアプリへ反映されます(地図上の位置やジャンルはそのまま維持されます)
- ⚠️ **新しい店の追加**: シートに新しい行(新しい店名)を追加すると、一覧・検索にはすぐ出てきますが、**地図上には表示されません**(住所からの座標取得や、料理ジャンルの分類は自動ではできないためです)。「⚠️新規追加分・地図未反映」という注記が付きます
- 🔄 新しく追加したお店を地図に反映したい場合は、私に「スプレッドシートを同期して」と声をかけてください。座標取得とジャンル分類をして、アプリ本体(`js/data.js`)に正式に組み込みます
- 🗑️ シートから行を削除すると、そのお店はアプリの表示からも消えます

## 5. 反映されないとき

- スプレッドシートの共有設定が「リンクを知っている全員が閲覧可」になっているか確認してください
- アプリ側は起動時に一度だけシートを読み込みます。編集後はアプリを再読み込み(リロード)してください
- 読み込みに失敗した場合は、自動的に最後にアプリへ組み込んだデータ(`js/data.js`)にフォールバックします(アプリが真っ白になることはありません)
